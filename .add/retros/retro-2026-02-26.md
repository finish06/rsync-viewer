## Agreed Changes

### 1. Enforce ruff format + check before every commit
Run `ruff format . && ruff check .` as a mandatory step before any `git commit`. No exceptions, even for docs-only commits. This prevents formatting drift that has been a recurring issue.

### 2. CI must pass before declaring work done (HARD GATE)
**This was agreed in retro-2026-02-24 but NOT enforced.** Making it non-negotiable:
- Before marking away tasks complete: run full test suite + ruff
- Before creating PRs: verify CI would pass locally
- Before suggesting merge: confirm all checks green
- If CI fails, fix before proceeding — do not defer

### 3. Proactive release iteration after milestone completion
After merging feature work to main, the system should:
- Prompt for a version bump + changelog update
- Releases must always be iterated when significant work lands on main
- Don't wait for the human to ask — suggest it proactively

- Away mode workflow — keep as-is
- Default recommendation pattern on questions — keep using
